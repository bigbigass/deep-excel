# DeepExcel Demo 前端精简中文版设计

**目标**

把前端所有页面统一成中文、减少非必要元素，并把 Demo 主线收敛为“上传文件 → 可见 AI 分析过程 → 生成报告”。

**用户定位**

- 面向客户演示，不是内部操作后台
- 强调能力展示，不强调系统运维信息
- 让客户能看懂 AI 在做什么，而不是只看到最终结果

**核心设计原则**

1. 首页只保留上传主入口和最少量说明
2. 分析页突出“AI 分析过程可见”，使用步骤 + 证据卡片表达
3. 报告页只保留交付结果、下载动作和简要说明
4. 全站去英文化，避免中英混排造成演示感割裂
5. 删除或弱化不适合客户演示的系统健康、运行时、操作说明等噪音信息

**页面调整**

## 首页

- 标题改为面向演示的中文表达
- 保留三步流程提示：上传数据、AI 分析、生成报告
- 保留上传组件作为唯一主动作
- 去掉上游模型健康检查
- 去掉过多能力卡片、英文标签和大段平台化描述

## 分析页

- 顶部改为中文任务概览
- 主区域先展示 AI 分析过程，再展示关键指标和图表证据
- AI 分析过程改成客户可理解的步骤说明：读取数据、识别异常、形成判断、生成报告
- 侧边保留任务状态，但移除偏技术的操作说明
- 所有标签、状态、提示均使用中文

## 报告页

- 改为简洁的交付结果页
- 保留报告编号、文件名、下载按钮、返回上传入口
- 用中文说明报告包含的内容和适合的使用场景
- 去掉英文眉标和多余展示块

**非目标**

- 不改后端接口协议
- 不增加样例双入口
- 不增加新的复杂交互流程
